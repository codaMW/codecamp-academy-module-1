Why Use let and const Instead of var in JavaScript

A modern JavaScript explanation for beginner developers.

What is ES6?

ES6 (ECMAScript 2015) is a major update to JavaScript that introduced many modern features to improve code readability, reliability, and scalability.

Some of the most important ES6 features include:

  • let and const
  • Arrow functions
  • Template literals
  • Classes
  • Modules
  • Destructuring

One of the most important improvements introduced in ES6 was better variable declaration using let and const.

The Problem with var

Before ES6, JavaScript only had var for declaring variables. However, var has several problems that often cause bugs in applications.

1. Scope Problems

var is function scoped, while let and const are block scoped.

Example using var


                    if (true) {
                        var message = "Hello";
                    }

                    console.log(message);

The variable is still accessible outside the block.

Example using let


                    if (true) {
                        let message = "Hello";
                    }

                    console.log(message); // ReferenceError

Because let respects block scope, the variable cannot be accessed outside.

2. Variable Redeclaration

With var, variables can be redeclared in the same scope. 


                    var name = "John";
                    var name = "Ana";

                    console.log(name);

This can overwrite variables and cause unexpected bugs.

Using let prevents this mistake: 


                    let name = "John";
                    let name = "Ana"; // Error

3. Hoisting Confusion

Variables declared with var are hoisted and initialized with undefined. 


                    console.log(age);
                    var age = 20;

This prints undefined, which can confuse developers.

With let: 


                    console.log(age);
                    let age = 20; // ReferenceError

This prevents accessing variables before they are declared.

4. Using const

const creates variables that cannot be reassigned. 


                    const PI = 3.1416;

                    PI = 3.14; // Error

This is useful for values that should never change. 


                    const API_URL = "https://api.example.com";
                    const MAX_USERS = 100;

Modern JavaScript Best Practice

Most professional developers follow this rule:

  • Use const by default
  • Use let if the value will change
  • Avoid var

                    const tax = 0.19;
                    let total = 100;

                    total = total + total * tax;

                    console.log(total);

Quick Comparison

Keyword Scope Reassign Redeclare
var Function Yes Yes
let Block Yes No
const Block No No

Conclusion

ES6 introduced let and const to make JavaScript safer, more predictable, and easier to maintain.

For modern JavaScript development, avoid var and prefer const or let.
